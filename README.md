# Supply Chain Intelligence GNN

Graph Neural Network for supply chain impact propagation.
Runs on **Intel Arc XPU** (B580) with **BF16** precision via PyTorch.

## Run Order

### 1 — Train

```bash
python train.py --csv \
  --companies data/examples/companies.csv \
  --edges     data/examples/edges.csv
```

Optional flags:

| Flag | Default | What it does |
|------|---------|--------------|
| `--epochs` | 300 | More = better convergence |
| `--lr` | 1e-3 | Learning rate |
| `--hidden` | 128 | Model width (try 256 for larger graphs) |
| `--layers` | 3 | GNN depth — more layers = deeper propagation |
| `--csv` | off | Use CSV files instead of synthetic data |
| `--companies` | data/examples/companies.csv | Path to your companies CSV |
| `--edges` | data/examples/edges.csv | Path to edges CSV |

### 2 — Fetch Live Data (optional but recommended)

Run this separately on your machine — requires internet:

```bash
python data_manager.py update           # fetch all tickers in companies.csv
python data_manager.py update NVDA TSM  # fetch specific tickers only
python data_manager.py status           # show cache with prices + ages
```

This writes `data/live_cache.json`. Restart `app.py` after updating to see live prices.

Special ticker mappings handled automatically:
- `000660` → `000660.KS` (SK Hynix on KRX)
- `SSNLF` → `005930.KS` (Samsung on KRX)
- `AIRB` → `AIR.PA` (Airbus on Paris)

### 3 — Start the UI

```bash
python app.py
```

### 4 — Open browser

```
http://localhost:5000
```

## UI Tabs

### Graph
Interactive force-directed graph of every company.
- **Node size** = GNN impact score
- **Node color** = sector (default) or impact (green/red)
- **Drag** nodes to rearrange, **scroll** to zoom, **click** background to deselect
- **Click any node** → detail panel slides up showing:
  - Revenue, margin, YoY growth, market cap, R&D%, CapEx%, debt ratio, GNN impact
  - Growth forecast (📌 company-specific or 📊 sector default)
  - Upstream suppliers with relationship strength bars — all clickable
  - Downstream customers with relationship strength bars — all clickable
- **Sidebar** = searchable company list sorted with impact pills
- **By Impact / By Sector** buttons toggle node coloring
- **Labels / Reset** buttons top-right

### Shock Propagation
Run what-if scenarios and see how growth signals ripple through the supply chain.

1. Pick mode: **By Sector** or **By Company**
2. Set a growth signal (−80% to +150%)
3. Hit **▶ Run Propagation**
4. Every company ranked by predicted Δ revenue impact with color-coded badges:
   - ⚡ **Direct** — companies in the shocked sector
   - ↑ **Supplier** — upstream suppliers of that sector
   - ↓ **Customer** — downstream customers
   - ~ **Indirect** — second-order effects
5. Click any result row to jump to that company in the graph

### Sector Bubbles
Force-directed bubble map where each bubble = one sector.
- **Bubble size** = total market cap of sector
- **Labels** show total market cap, avg GNN impact, sector forecast %
- **Lines** between bubbles = real cross-sector supply chain links from edges.csv
- **Drag** bubbles to rearrange, **scroll/pinch** to zoom, same as company graph
- **Click any bubble** → detail panel showing:
  - Total market cap + revenue, avg GNN impact, sector forecast
  - All companies sorted by market cap, each clickable to open in graph

### Investment Calculator
Project returns for companies or sectors over a custom time horizon.

**Standard mode** — clean card view per company showing annual growth, invested, final value, net gain.

**Detailed mode** — adds a second row of fundamentals per card. Switch filter to **Single Company** for the full deep-dive:
- Supply Chain Risk score (0–10) based on connectivity
- Sector Exposure bar chart — which sectors your supply chain touches and by how much market cap
- Upstream Suppliers (2-hop) — suppliers of your suppliers with forecasts + GNN impact
- Downstream Customers (2-hop) — customers of your customers

Filter modes: **All Companies**, **By Sector**, **Single Company**

Scenarios: Use Growth Forecasts · Bull +30% · Base +15% · Bear −10% · Custom %

Presets: AI Infra · EV Transition · Memory Boom · Top Impact

Clicking any company card jumps to the graph view for that company.

### 📡 Live Data
Real-time stock prices and quarterly financials (requires `data_manager.py` to have been run).

**Stock table** — all 59 companies with:
- Live price + change% (green/red)
- 30-day sparkline trend chart
- Market cap, margin, YoY growth, sector forecast, GNN impact
- Sortable by any column (click header)
- Filterable by sector and text search

**Company detail panel** (click any row):
- Live price with 52-week range bar showing current position
- Latest quarterly income statement (revenue, gross profit, operating income, net income)
- 30-day trend chart
- Buttons: View in Graph, Analyse Investment

**Add / Update Company** panel:
- Drag-and-drop or click to upload a CSV (use `companies_template.csv` format)
- Preview what will be added before submitting
- Add supply chain edges with the inline edge form
- Handles both new companies and updates to existing ones

**Update Company Data** panel:
- Pick any company from dropdown
- Fill in only the fields you want to change
- Writes directly to `companies.csv`
- Retrain after saving to apply to GNN predictions

## edges.csv — Column Reference

Each row = one directional supply chain relationship: **supplier → customer**

```
COLUMN LAYOUTS (auto-detected, pick one):
  supplier_ticker + customer_ticker     ← recommended
  supplier_name   + customer_name
  supplier_id     + customer_id         (0-based row index from companies.csv)

OPTIONAL:
  relationship_strength    0.0 – 1.0   how critical is this link
                           Also accepted as 0–100 (auto-scaled)
                           Also accepted column names: strength, weight
                           Defaults to 1.0 if omitted
```

Lines starting with `#` are treated as comments and ignored.

## How to Update companies.csv

### Adding a new company

Append a row to `companies.csv`:
```csv
xAI,XAI,Software,2.0,0%,0.1,200%,50.0,30%,2%,60%,12
```

Add its supply relationships to `edges.csv`:
```csv
NVDA,XAI,0.80
XAI,MSFT,0.20
```

Retrain:
```bash
python train.py --csv \
  --companies data/examples/companies.csv \
  --edges     data/examples/edges.csv
```

Or use the **Add / Update Company** panel in the Live Data tab to do it through the UI.

### Updating financials

Edit the row directly in `companies.csv`, or use the **Update Company Data** panel in the UI, then retrain.

### Changing a relationship strength

Find the row in `edges.csv`, update the third column (0.0 = weak, 1.0 = critical), retrain.

### After any change

Always retrain before restarting the UI:
```bash
python train.py --csv \
  --companies data/examples/companies.csv \
  --edges     data/examples/edges.csv
python app.py
```

## GNN Architecture

```
SupplyChainGNN
│
├── Encoder              linear(feat_dim + 1 → hidden)   node features + shock concatenated
│
├── Upstream GCN         3× GraphConvLayer   propagates toward suppliers
├── Downstream GCN       3× GraphConvLayer   propagates toward customers
│
├── Shock diffusion      GraphConvLayer × 2  explicitly spreads shock to neighbours
│                        (upstream pass + downstream pass, summed)
│
└── Output head          linear(hidden×4 → hidden×2 → hidden → 1)
                         → predicted revenue impact %
```

Node features (20 dimensions):
- Revenue, margin, debt ratio, YoY growth, market cap
- R&D%, CapEx%, supplier count, customer count, growth forecast
- Sector one-hot (10 sectors)

The shock vector is concatenated to node features before encoding — so it propagates through all graph convolution layers rather than being injected only at the end. A separate shock diffusion stream explicitly spreads the shock signal to neighbours via the adjacency matrix, then combines with the main stream for the final prediction.

BF16 activates automatically on XPU or CUDA. Falls back to FP32 on CPU.

Key supply chain paths:
- `ASML → TSM → NVDA/AMD → hyperscalers` (AI chip stack)

- `ARM → QCOM/NVDA/MRVL → auto/cloud` (IP licensing)
- `WOLF → ON/NXPI → TSLA/TM` (SiC power electronics for EV)
- `FCX → NUE → CAT/HON` (copper → steel → industrials)
- `INTC → DELL/HPE/SMCI/TSLA/TM` (x86 server + automotive)
