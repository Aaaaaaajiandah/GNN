# Supply Chain Intelligence GNN

Graph Neural Network for supply chain impact propagation.
Runs on **Intel Arc XPU** (B580) with **BF16** precision via PyTorch. Data for intel (INTC) is also currently wrong, will be updated later.

---

## File Structure

```
GNN/
│
├── app.py                  ← Flask web server (run this to see the UI)
├── train.py                ← Train the GNN model
├── companies.csv           ← Your company data (edit this to add/update companies)
├── companies_template.csv  ← Reference showing every accepted column format
├── edges.csv               ← Your supply chain relationships (who supplies who)
├── edges_template.csv      ← Reference showing edge column formats
├── README.md               ← This file
│
├── data/
│   ├── dataset.py          ← Data structures, feature engineering, graph builder
│   └── csv_loader.py       ← CSV loader (drop-in replacement for synthetic data)
│
├── models/
│   └── gnn.py              ← GNN architecture (GraphConvLayer, SupplyChainGNN)
│
└── templates/
    └── index.html          ← Dashboard UI (opened in your browser)
```

After you train, two more files appear automatically:

```
├── data/
│   └── graph.json          ← Processed graph (generated by train.py, read by app.py)
└── models/
    └── checkpoint.pt       ← Trained model weights (generated by train.py)
```

## Run Order

### 1 — Train

```bash
python train.py --csv --companies companies.csv --edges edges.csv
```

This reads your CSVs, builds the graph, trains on XPU with BF16,
and saves `models/checkpoint.pt` + `data/graph.json`.

Optional flags:

| Flag | Default | What it does |
|------|---------|--------------|
| `--epochs` | 300 | More = better convergence, takes longer |
| `--lr` | 1e-3 | Learning rate |
| `--hidden` | 128 | Model width (try 256 for larger graphs) |
| `--layers` | 3 | GNN depth — more layers = deeper propagation |
| `--csv` | off | Use CSV files instead of synthetic data |
| `--companies` | data/examples/companies.csv | Path to companies CSV |
| `--edges` | data/examples/edges.csv | Path to edges CSV |

### 2 — Start the UI

```bash
python app.py
```

### 3 — Open browser

```
http://localhost:5000
```

## The UI

### Graph tab
Interactive force-directed graph of every company.
- **Node size** = GNN impact score
- **Node color** = sector
- **Click any node** → slides up a detail panel showing:
  - Revenue, margin, YoY growth, market cap, R&D%, CapEx%, debt ratio
  - Upstream suppliers with relationship strength bars
  - Downstream customers with relationship strength bars
  - Each supplier/customer is clickable to navigate the chain
- **Sidebar** = searchable company list, sorted with impact scores
- **Labels / Reset** buttons top-right

### Shock Propagation tab
Run a what-if scenario:
1. Pick a sector (e.g. Semiconductors)
2. Set a growth signal (e.g. +50%)
3. Hit **Run Propagation**
4. Every company in the graph is ranked by predicted Δ revenue impact
5. Click any result row to jump to that company in the graph

This is how you answer questions like:
- "If AI datacenter demand grows 60%, how much does that flow back to ASML?"
- "If automotive production drops 20%, which tier-2 suppliers get hit hardest?"

### Sectors tab
Overview card for each sector showing avg GNN impact and all company tickers.
Click any ticker to open that company's detail panel.

## companies.csv — Column Reference

```
REQUIRED:
  name      Full company name       e.g.  NVIDIA
  ticker    Stock ticker            e.g.  NVDA
  sector    Industry sector         e.g.  Semiconductors

OPTIONAL (leave blank = 0):
  revenue_bn              Annual revenue in $B
  margin                  Net profit margin
  debt_ratio              Total debt / equity  (raw ratio, not %)
  yoy_growth              Year-over-year revenue growth
  market_cap_bn           Market cap in $B
  r_and_d_pct             R&D as % of revenue
  capex_pct               CapEx as % of revenue
  sector_growth_forecast  Analyst growth forecast for this sector
  forecast_horizon_months Forecast period in months
```

Accepted sectors:
`Semiconductors` `Automotive` `Aerospace` `Energy` `Retail`
`Healthcare` `Financials` `Materials` `Software` `Industrials`

Anything close fuzzy-maps automatically (e.g. `chip` → Semiconductors,
`pharma` → Healthcare, `bank` → Financials).

Percentage columns accept any of these — all mean the same thing:
```
0.385    38.5    38.5%
```

Revenue / market cap columns accept suffixes:
```
69.3    69.3B    69300M    0.0693T
```

## edges.csv — Column Reference

Each row = one directional relationship: **supplier → customer**
(supplier sells TO customer).

```
REQUIRED (pick one layout — auto-detected):
  supplier_ticker + customer_ticker     ← recommended
  supplier_name   + customer_name
  supplier_id     + customer_id         (0-based row number from companies.csv)

OPTIONAL:
  relationship_strength    0.0 – 1.0  (how critical is this link)
                           Also accepted as 0–100, auto-scaled
                           Also accepted column names: strength, weight
                           Defaults to 1.0 if omitted
```

Lines starting with `#` are treated as comments and ignored, so you can
section your edges file like the included example does.

## How to Update companies.csv

### Adding a new company

Open `companies.csv` and add a row at the bottom:

```csv
Super Micro Computer,SMCI,Industrials,14.9,8.2%,0.42,110.8%,28.0,2.5%,4.8%,,
```

Then add its supply relationships to `edges.csv`:

```csv
NVDA,SMCI,0.80
MU,SMCI,0.50
SMCI,MSFT,0.35
```

Then retrain:
```bash
python train.py --csv --companies companies.csv --edges edges.csv
```

### Updating an existing company's financials

Just edit the row in `companies.csv` directly — change the numbers,
save, retrain. The ticker is the key so don't change that unless you
also update edges.csv to match.

### Adding a sector growth forecast

Add or update the last two columns on any company row:

```csv
NVDA,NVDA,Semiconductors,...,...,...,60%,12
```

This means: "analysts expect the Semiconductors sector to grow 60%
over the next 12 months." The GNN uses this as the shock starting
signal for that company when you run propagation.

### Removing a company

Delete its row from `companies.csv` and delete all rows in `edges.csv`
that reference its ticker. Then retrain.

### Changing a supply relationship strength

Find the row in `edges.csv` and update the third column:

```csv
TSM,NVDA,0.95     ← change this number (0.0 = weak link, 1.0 = critical dependency)
```

Then retrain.

### After any CSV change

Always retrain before restarting the UI:
```bash
python train.py --csv --companies companies.csv --edges edges.csv
python app.py
```

`app.py` loads `data/graph.json` and `models/checkpoint.pt` at startup —
both are regenerated by `train.py`.

## Architecture

```
SupplyChainGNN
│
├── Encoder          linear(feat_dim → hidden)
│
├── Upstream GCN     3× GraphConvLayer   propagates toward suppliers
├── Downstream GCN   3× GraphConvLayer   propagates toward customers
│
├── Shock projection linear(1 → hidden)  injects sector growth signal
│
└── Output head      linear(hidden×3 → 1)  → predicted revenue impact
```

Node features (20 dimensions per company):
- Revenue, margin, debt ratio, YoY growth, market cap
- R&D%, CapEx%, supplier count, customer count, growth forecast
- Sector one-hot (10 sectors)

BF16 activates automatically on XPU or CUDA. Falls back to FP32 on CPU.

- `FCX → NUE → CAT/HON` (copper → steel → industrials)
